# 다담 (Dadam) — 하루를 담다

캘린더 · 할 일 · 마감 · 메모 · 일지를 한 곳에 담는 개인용 웹 앱.
순수 HTML/CSS/JavaScript(바닐라)로 만들어졌고, 데이터는 브라우저 **localStorage**에 저장되며 선택적으로 **Supabase 클라우드**로 여러 기기와 동기화된다.

> 사용 목적: 대학원 일정 관리, 하루 할 일 정리, 프로젝트/논문 마감 추적, 정보 메모 취합, 하루 일지.
>
> 배포: **https://yunhocha.github.io/dadam/** (GitHub Pages) · 저장소: `github.com/YunhoCha/dadam`

---

## 실행 방법

빌드 과정 없이 정적 파일이다.

```
# 방법 1) 그냥 index.html을 브라우저로 열기
# 방법 2) 간단한 정적 서버 (예: 파이썬)
python -m http.server 4321   # → http://localhost:4321
# 방법 3) 미리보기(이 저장소): .claude/serve.ps1 (PowerShell 정적 서버, 포트 4321)
```

배포는 GitHub에 push → GitHub Pages 자동 반영. 서비스워커가 새 버전을 감지하면 **자동 새로고침**한다.

## 레이아웃

```
┌──────────────────────────────┬────────────────────┐
│ 헤더(브랜드·달이동·💾백업·통합검색) │  [탭] 일정·할일 / 메모 │
│ 카테고리 범례(가로, 필터 겸용)    │ ┌────────────────┐ │
│ ┌────────────────────────┐   │ │ ⏰ 다가오는 마감   │ │
│ │       월간 캘린더         │   │ │ 📅 선택 날짜 할 일 │ │
│ │  (마감칩·할일칩·기간바·📝)  │   │ │ 📔 일지 + 기분     │ │
│ │                        │   │ │ 📥 INBOX/주간/월간 │ │
│ └────────────────────────┘   │ └────────────────┘ │
└──────────────────────────────┴────────────────────┘
```

좁은 폭(≤860px)에서는 캘린더 → 패널 순으로 세로로 쌓이며 페이지 전체가 스크롤된다. 우측 할 일 패널을 넓게(460px) 배치.

## 기능 정리

### 캘린더
- 월간 보기(달에 맞춰 4~6주 자동, 보통 5주), 이전/다음/오늘 이동, 일·토 색 구분, 오늘·선택일 강조.
- 위쪽에 **카테고리 범례**(가로 바, 클릭해 필터). 설정 ⚙로 카테고리 관리.
- 날짜 클릭 → 우측에 그 날의 할 일/일지 표시. 날짜 칸 **＋ 버튼 → 상세창**으로 일정 추가.
- **빈 영역을 드래그**해서 여러 날에 걸친 **기간 작업** 생성(시작~종료 바로 표시).
- **할 일 칩을 다른 날짜로 드래그**해서 이동(반복·기간 제외). **Ctrl/⌘ 누른 채 드래그하면 복사**. **INBOX 블럭도 달력으로 드래그** 가능.
- 빈 날짜 칸 **더블클릭 → 빠른 할 일 추가**(우측 패널에 바로 입력).
- 칩 구조: **왼쪽 완료 체크박스 / 가운데 내용 / 오른쪽 카테고리 색 점·📝 메모 아이콘**. 칩 클릭 → 상세 모달.
- 날짜 칸에 **마감 칩**(D-day + 제목)과 **연결된 메모 📝 아이콘** 표시.

### 할 일 / 일정
- 노션식 블럭(체크박스 + 인라인 편집), `Enter`로 추가 / 빈 블럭 `Backspace`로 삭제.
- 칩 클릭 또는 ⚙ → 상세 모달: **기간(시작~종료일)**, **시작·종료 시간**, **카테고리(태그처럼 복수 지정)**, **별표(중요)**, **반복(매주/격주 + 종료일)**, **작업별 노트(메모)**.
- **작업 노트**가 있으면 캘린더 칩·우측 블록에 📝 아이콘.
- **하위 단계(서브태스크)**: 상세 모달 또는 우측 블록 아래에서 단계 추가·체크·편집·삭제. 진행도 `☑ 2/3` 표시.
- 정렬: 기간 작업 → 별표 → 시간 순.
- **인라인 문법(라이브)**:
  - `#` 입력 → **카테고리 드롭다운**(필터·키보드/클릭·없으면 새로 생성). 블럭은 색 칩으로, 상세창은 카테고리 버튼 하이라이트로 피드백. (블럭 입력칸·상세창 제목 모두 지원)
  - `12시·12:30·12시30분·오후 3시·10시반` 등 → **시간 자동 설정**(스페이스 시 블럭 플래시). 해당 텍스트는 정리됨.

### 마감 / 데드라인
- ＋로 등록: 제목, 마감일, 카테고리, **진행 단계(서브태스크)** 체크.
- **자동 D-day**(D-7 / D-DAY / D+n), 임박(3일 이내) 빨강 강조.
- "다가오는 마감" 가까운 순 Top 5, 캘린더에도 제목 칩으로 표시.

### 메모
- 별도 탭. 검색(제목·본문), **고정(핀)**, 카드 목록(고정 → 최근수정 순).
- 카드 클릭 → **모달 편집창**에서 작성. 서식 툴바: **굵게 / 목록 / 체크박스 / 링크 / 이미지**.
- 이미지: 버튼 또는 **붙여넣기(Ctrl+V)** 로 인라인 삽입(dataURL).
- **날짜 연결**(📅): 특정 날짜에 메모를 붙이면 그 날짜 칸에 📝 아이콘. 여러 개면 클릭 시 목록 팝업.

### 일지
- 날짜별 자유 작성란(자동 저장), **기분 5단계**(😣🙁😐🙂😄), **회고 템플릿** 1클릭 삽입.

### 빠른 메모 블럭 (INBOX / 주간 / 월간)
- 우측 패널 하단의 노션식 블럭 입력칸. `Enter`로 블럭 추가.
- **INBOX**: 아무거나 임시로. **주간/월간**: 선택한 날짜의 주·월에 연동.
- 블럭 핸들(⠿)을 **달력으로 드래그** → 그 날짜의 할 일로 이동(드롭 시 #·시간 인라인 문법도 적용).
- 📝 버튼 → **메모로 보내기**(메모 편집창이 열림).

### 통합 검색 / 우측 패널 탭
- 헤더 검색창에서 **일정·할 일·마감·메모·일지**를 한 번에 검색, 클릭 시 이동/열기.
- 우측 패널: **🗓️ 일정·할 일** / **📝 메모** 두 탭.

### 백업 / 복원
- 헤더 **💾** → 전체 데이터를 **JSON 파일로 내보내기**, 백업 파일에서 **가져오기(복원)**. 복원 시 클라우드에도 반영.

## 기기 간 동기화

- Supabase에 전체 상태(JSON)를 한 칸(jsonb)에 저장해 **여러 기기·모바일에서 같은 계정으로 동기화**.
- 이메일 **매직링크** 로그인(비밀번호 없음), 다른 기기 변경 **실시간 반영**, 마지막 저장 우선.
- RLS로 사용자별 격리(코드의 `Publishable` 키는 공개돼도 안전). 미설정 시 **로컬 전용**으로 동작.
- 설정·배포 방법: [`SYNC_SETUP.md`](SYNC_SETUP.md) · 구조 설명: [`원리설명.md`](원리설명.md).

## 데이터 모델 (localStorage 키: `dadam.v2`)

```js
{
  categories: [   // 사용자 정의 카테고리
    { id, label, color }   // color: 팔레트 키(sky/mint/lilac/coral/sun/rose/grass/ocean/grape/slate)
  ],
  items: [        // 일정/할 일
    { id, text, date:"YYYY-MM-DD", endDate:null|"YYYY-MM-DD",  // endDate 있으면 기간 작업
      start, end, categories:[catId,...], note, subtasks:[{id,text,done}], star,
      done, doneDates:{}, recur:null|{freq:"weekly"|"biweekly", until} }
  ],
  deadlines: [ { id, title, date, category, subtasks:[{id,text,done}] } ],
  notes:     [ { id, title, body(HTML), pinned, updated, linkDate } ],
  journals:  { "YYYY-MM-DD": { mood:0-5, text(HTML) } },
  inbox:     [ {id,text} ],              // 임시 메모 블럭
  weekly:    { "일요일키": [{id,text}] }, // 주 단위 블럭(날짜 연동)
  monthly:   { "YYYY-MM": [{id,text}] }, // 월 단위 블럭(날짜 연동)
  _updatedAt                              // 동기화 충돌 비교용 타임스탬프
}
```

- 반복 일정은 원본 1개를 저장하고 캘린더 렌더 시 규칙에 맞는 날짜에 투영. 반복 완료는 `doneDates[날짜]`에 저장.
- 기간 작업(`endDate`)은 시작~종료 사이 각 날짜에 바 형태로 표시.
- 옛 형식 데이터는 로드 시 자동 정규화: `dadam.v1`→v2, 단일 `category`→`categories` 배열, 누락 필드(`note`·`endDate`·`inbox`/`weekly`/`monthly`) 기본값 채움. 로컬·클라우드 모두 동일 처리.

## 업데이트 내역 (최근)

**클라우드·배포**
- Supabase 클라우드 동기화(이메일 매직링크 로그인, 실시간), 마지막 저장 우선.
- GitHub Pages 배포 + PWA(홈 화면 설치·오프라인). 서비스워커 **자동 업데이트**(HTTP 캐시 우회).
- **💾 JSON 백업/복원** 기능.

**카테고리**
- 카테고리를 **사용자 설정**(추가·이름·색·삭제, 10색 팔레트)으로 전환, state에 저장.
- 할 일에 카테고리를 **태그처럼 복수 지정**. 캘린더 위쪽 가로 **범례/필터**.

**캘린더 / 할 일**
- **기간(여러 날) 작업** + **날짜 드래그로 생성**.
- 칩 재배치: **왼쪽 완료 체크박스 / 오른쪽 카테고리 점·📝**. 칩 클릭 → **상세 모달**.
- 날짜 칸 **＋ → 상세창**. 5주 기본 보기, 우측 패널 확대.
- **작업별 노트(메모)** + 📝 아이콘.
- **인라인 문법**: `#카테고리` 드롭다운(라이브, 블럭·상세창), `12시/12:30/오후 3시` 시간 자동 인식·플래시.

**기타**
- **INBOX/주간/월간** 빠른 메모 블럭(드래그→달력, 📝→메모).
- 글자·아이콘 전체 확대, 색·굵기 진하게.
