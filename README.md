# 다담 (Dadam) — 하루를 담다

캘린더 · 할 일 · 마감 · 메모 · 일지를 한 곳에 담는 개인용 웹 앱.
순수 HTML/CSS/JavaScript(바닐라)로 만들어졌고, 모든 데이터는 브라우저 **localStorage**에 저장된다.

> 사용 목적: 대학원 일정 관리, 하루 할 일 정리, 프로젝트/논문 마감 추적, 정보 메모 취합, 하루 일지.

---

## 실행 방법

빌드 과정 없이 정적 파일이다.

```
# 방법 1) 그냥 index.html을 브라우저로 열기
# 방법 2) 간단한 정적 서버
python -m http.server 4321
# → http://localhost:4321
```

## 디자인

- **여름 하늘 팔레트**: 연한 하늘색 그라데이션 배경, 반투명 유리(글래스) 패널, 부드러운 파란 그림자.
- 제목 폰트: `Gowun Dodum` (손글씨 느낌) / 본문: `Noto Sans KR`.
- 카테고리: 기본 5종(수업·세미나·미팅·개인·기타) + **설정(⚙)에서 직접 추가·이름변경·색 변경·삭제** 가능. 색 팔레트 10종.


## 기능 정리

### 캘린더
- 월간 보기(달에 맞춰 4~6주 자동, 보통 5주), 이전/다음/오늘 이동, 일·토 색 구분, 오늘·선택일 강조.
- 왼쪽에 **카테고리 범례**(클릭해 필터). 우측 할 일 패널을 넓게 배치.
- 날짜 클릭 → 우측에 그 날의 할 일/일지 표시.
- **할 일 칩을 다른 날짜로 드래그**해서 이동(반복 일정 제외).
- 날짜 칸에 **마감 칩**(D-day + 제목)과 **연결된 메모 📝 아이콘** 표시.

### 할 일 / 일정
- 노션식 블럭(체크박스 + 인라인 편집), `Enter`로 추가 / 빈 블럭 `Backspace`로 삭제.
- ⚙ 상세 모달: **시작·종료 시간**, **카테고리(태그처럼 복수 지정)**, **별표(중요)**, **반복(매주/격주 + 종료일)**.
- 정렬: 별표 → 시간 순 자동.
- 칩에 시간/별표/반복(↻) 표시, 완료율 원형 링.

### 마감 / 데드라인
- ＋로 등록: 제목, 마감일, 카테고리, **진행 단계(서브태스크)** 체크.
- **자동 D-day**(D-7 / D-DAY / D+n), 임박(3일 이내) 빨강 강조.
- "다가오는 마감" 가까운 순 Top 5, 캘린더에도 제목 칩으로 표시.

### 메모
- 별도 탭. 검색(제목·본문), **고정(핀)**, 카드 목록(고정 → 최근수정 순).
- 카드 클릭 → **모달 편집창**(별도 창)에서 작성.
- 서식 툴바: **굵게 / 목록 / 체크박스 / 링크 / 이미지**.
- 이미지: 버튼 또는 **붙여넣기(Ctrl+V)** 로 인라인 삽입(dataURL).
- **날짜 연결**(📅): 특정 날짜에 메모를 붙이면 그 날짜 칸에 📝 아이콘. 같은 날짜에 메모가 여러 개면 클릭 시 목록 팝업.

### 일지
- 날짜별 자유 작성란(자동 저장).
- **기분 5단계** 이모지(😣🙁😐🙂😄).
- **회고 템플릿**: `오늘 한 일 / 배운 것 / 내일 할 일` 한 번에 삽입.

### 통합 검색
- 헤더 검색창에서 **일정·할 일·마감·메모·일지**를 한 번에 검색.
- 결과 클릭 시 해당 날짜로 이동하거나 메모를 연다.

### 우측 패널 탭
- **🗓️ 일정·할 일**(마감+할일+일지) / **📝 메모** 두 탭으로 분리, 각자 스크롤.

## 데이터 모델 (localStorage 키: `dadam.v2`)

```js
{
  categories: [   // 사용자 정의 카테고리
    { id, label, color }   // color: 팔레트 키(sky/mint/lilac/coral/sun/rose/grass/ocean/grape/slate)
  ],
  items: [        // 일정/할 일
    { id, text, date:"YYYY-MM-DD", start, end, categories:[catId,...], star,
      done, doneDates:{}, recur:null|{freq:"weekly"|"biweekly", until} }
  ],
  deadlines: [    // 마감
    { id, title, date, category, subtasks:[{id,text,done}] }
  ],
  notes: [        // 메모
    { id, title, body(HTML), pinned, updated, linkDate }
  ],
  journals: {     // 날짜별 일지
    "YYYY-MM-DD": { mood:0-5, text(HTML) }
  }
}
```

- 반복 일정은 원본 1개를 저장하고, 캘린더 렌더 시 규칙에 맞는 날짜에 투영(occurrence)한다. 반복 완료 상태는 `doneDates[날짜]`에 저장.
- `dadam.v1`(초기 버전) 데이터는 자동으로 v2로 마이그레이션된다.
- 단일 `category` 형식의 기존 데이터(로컬·클라우드 모두)는 로드 시 `categories` 배열로 자동 변환된다.

## 기기 간 동기화 (신규)

- Supabase에 전체 상태(JSON)를 저장해 **여러 기기·모바일에서 같은 계정으로 동기화**.
- 이메일 **매직링크** 로그인(비밀번호 없음), 다른 기기 변경 **실시간 반영**.
- 미설정 시 기존처럼 **로컬 전용**으로 동작. 설정 방법은 [`SYNC_SETUP.md`](SYNC_SETUP.md).
