# 다담 동기화 — 원리 설명

> 여러 기기에서 같은 데이터를 보는 구조가 어떻게 동작하는지 정리한 문서.

핵심 한 줄: **로그인한 상태에서는 모든 변경이 자동으로 내 계정(클라우드)에 저장된다.**

---

## 1. 등장하는 3개의 서비스 — 각자 역할이 다름

| 서비스 | 역할 | 비유 |
|--------|------|------|
| **GitHub** | 앱 **파일(코드) 보관 + 웹주소 제공** (`yunhocha.github.io/dadam`) | 앱이 설치된 **건물 주소** |
| **Supabase** | **데이터 저장 + 로그인(인증)** 처리 | 내 **개인 금고 + 출입증 발급소** |
| **이메일** | 로그인 **신원 확인** (매직링크 받는 곳) | 금고를 여는 **본인 확인 열쇠** |

셋은 분리돼 있다. GitHub엔 **코드만**, Supabase엔 **내 데이터만** 있다.
GitHub은 일정/메모 내용을 전혀 모른다.

---

## 2. 데이터는 두 군데에 저장된다 (핵심 원리)

```
   [내 기기]                          [클라우드]
 ┌──────────────┐                ┌────────────────────┐
 │ localStorage │  ←── 동기화 ──→ │  Supabase DB        │
 │ (브라우저 캐시) │                │  user_data 테이블    │
 │  = 빠른 사본   │                │  = 진짜 원본(영구)    │
 └──────────────┘                └────────────────────┘
   기기마다 따로                      내 계정에 1개
```

- **localStorage**: 각 기기/브라우저에 있는 **빠른 사본**. 인터넷이 없어도 즉시 뜨게 해준다.
- **Supabase**: **진짜 원본**. 내 계정(`user_id`) 아래 **JSON 한 덩어리**로 저장된다.
  기기를 바꿔도, 캐시를 지워도, 새 기기를 사도 → 로그인하면 여기서 다시 받아온다.

전체 데이터(일정·할 일·마감·메모·일지)가 **딱 한 칸(jsonb)에 통째로** 들어간다.
그래서 구조가 단순하고 빠르다.

### 데이터 모델
`user_data` 테이블 한 줄:

| 컬럼 | 내용 |
|------|------|
| `user_id` | 내 계정 식별자(로그인한 사람) |
| `data` | 다담 전체 상태 JSON (`{ items, deadlines, notes, journals, _updatedAt }`) |
| `updated_at` | 마지막 저장 시각 |

## 3. 실제로 일어나는 과정 (3가지 순간)

### ① 앱을 켤 때
```
캐시(localStorage) 즉시 표시 → 클라우드에서 최신본 받아옴 → 더 최신이면 교체
```
켜자마자 빠르게 뜨고, 곧 최신 상태로 맞춰진다.

### ② 뭔가 편집할 때 (할 일 추가, 메모 수정 등)
```
화면 반영 → localStorage 저장 → 1.2초 후 클라우드로 자동 업로드 (🟠 → 🟢)
```
저장 버튼을 누를 필요 없음. 자동이다. (1.2초 디바운스 = 연속 입력을 묶어서 한 번에 올림)

### ③ 다른 기기에서 바꿨을 때
```
폰에서 수정 → 클라우드 갱신 → PC가 실시간으로 감지 → 자동 반영
```
같은 계정이면 기기끼리 알아서 맞춰진다. (Supabase Realtime 구독)

---

## 4. 충돌 · 오프라인 처리

- **충돌**: 두 기기에서 동시에 다른 내용을 고치면 **나중에 저장된 쪽이 이긴다**(마지막 저장 우선, `_updatedAt` 비교). 혼자 쓰는 용도라 보통 문제 없다.
- **오프라인**: 인터넷이 끊겨도 localStorage에 저장돼 계속 작동 → 다시 연결되면 다음 편집 때 올라간다.

---

## 5. 보안 & 비용

- **보안**: Supabase의 **RLS(행 수준 보안)** 정책 때문에, 로그인한 본인만 본인 데이터에 접근 가능.
  코드에 박힌 `Publishable` 키는 **공개돼도 안전한 키**라 GitHub에 올라가도 괜찮다.
  (반대로 `Secret` 키는 절대 올리면 안 됨 — 이 프로젝트엔 사용하지 않음.)
- **비용**: GitHub Pages, Supabase 둘 다 **무료 요금제**로 충분.

---

## 6. 꼭 알아둘 점

1. **로그인을 해야 동기화된다.** 로그인 안 한 기기는 그 기기 localStorage에만 저장된다.
   새 기기에선 항상 로그인부터.
2. **Supabase 무료 프로젝트는 약 7일간 아무 접속이 없으면 "일시정지"** 된다.
   데이터는 지워지지 않고, 대시보드에서 클릭 한 번으로 다시 켜진다.
   평소 앱을 쓰면 자동으로 살아있는 상태가 유지된다.
3. 정말 안전하게 백업하려면 가끔 데이터를 **JSON으로 내보내기** 하는 것을 추천.
